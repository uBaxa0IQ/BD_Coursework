# План аналитической части

## 1.1. Анализ предметной области

- Проблема моделирования световых эффектов гирлянды
- Требования к точности и визуализации
- Область применения (проектирование декоративного освещения, виртуальные инсталляции, демонстрация принципов геометрической оптики)
- Теоретические основы:
  - Геометрическая оптика
  - Преломление света (Закон Снеллиуса)
  - Диффузное отражение (Модель Ламберта)
  - Трассировка лучей как метод моделирования света
  - Предварительный расчет освещения (baking)

## 1.2. Обзор существующих аналогов

- 1.2.1. DIALux evo
  - Отраслевой стандарт для светотехнического проектирования
  - Расчет освещенности рабочих поверхностей
  - Ориентация на стандартные осветительные приборы
  - Ограничения для процедурной генерации каустических эффектов
  - Плюсы: профессиональный инструмент, точные расчеты
  - Минусы: не подходит для моделирования декоративных гирлянд, высокая стоимость

- 1.2.2. Blender (Cycles)
  - Универсальный пакет 3D-моделирования
  - Метод трассировки путей
  - Поддержка предварительного расчета освещения
  - Высокий порог вхождения, значительные вычислительные ресурсы
  - Плюсы: универсальность, физическая корректность
  - Минусы: избыточность для задачи, сложность использования

- 1.2.3. Unreal Engine 5
  - Игровой движок с передовыми технологиями работы со светом
  - Система предварительного расчета освещения
  - Избыточность для задачи моделирования одной гирлянды
  - Плюсы: передовые технологии, интерактивность
  - Минусы: избыточность, сложность настройки

- 1.2.4. Mitsuba 3
  - Исследовательская система рендеринга
  - Эталон физической корректности
  - Управление через скрипты, отсутствие удобного GUI
  - Плюсы: физическая корректность, открытый исходный код
  - Минусы: отсутствие GUI, сложность использования

- 1.2.5. LuxCoreRender
  - Рендерер с открытым исходным кодом
  - Двунаправленная трассировка путей
  - Низкая скорость работы (предварительный расчет)
  - Плюсы: открытый исходный код, физическая корректность
  - Минусы: низкая скорость, отсутствие специализации

- 1.2.6. Сравнительный анализ аналогов
  - Таблица с критериями сравнения:
- Метод расчета (моделирования света)
- Специализация
- Поддержка предварительного расчета
- Режим работы (интерактивный / предварительный расчет / реальное время)
    - Стоимость
    - Простота использования
  - Сравниваемые системы: DIALux evo, Blender (Cycles), Unreal Engine, Mitsuba 3, LuxCoreRender, разрабатываемая система
  - Вывод: почему существующие решения не подходят и почему нужно писать своё (ваше будет быстрее, дешевле, с уникальной функцией)

## 1.3. Постановка задачи

- Формальное описание того, что должна делать программа:
  - Входные данные:
    - Конфигурация источников света (позиция, интенсивность, цвет)
    - Параметры защитных колпачков (радиус, коэффициент преломления, матовость, цветовой фильтр)
    - Геометрия сцены (стена, сферы)
    - Параметры камеры (позиция, направление, разрешение)
  - Выходные данные:
    - Карты освещенности (lightmap) для поверхностей
    - Изображение вида камеры
  - Ограничения:
    - Визуализация только эффектов освещения на поверхностях
    - Расчет для статической конфигурации источников света
    - Ограничения на количество источников и объектов сцены (определяются производительностью)
- Цель работы: создание программной системы для моделирования световых паттернов статических декоративных гирлянд
- Требования к системе:
  - Моделирование прохождения света через преломляющие среды
  - Формирование карт освещенности на диффузных поверхностях
  - Поддержка стохастического рассеивания
  - Интерактивный просмотр сцены
  - Графический пользовательский интерфейс
  - Инструменты создания паттернов (точка, линия, дуга, круг)
  - Редактирование параметров источников и колпачков
  - Интерактивное управление камерой
  - Визуализация в реальном времени
