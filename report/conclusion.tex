\chapter*{ЗАКЛЮЧЕНИЕ}
\addcontentsline{toc}{chapter}{ЗАКЛЮЧЕНИЕ}

В рамках выполнения курсовой работы была успешно спроектирована
и реализована база данных для хранения и анализа спортивной статистики.
Поставленная цель достигнута в полном объёме посредством решения
сформулированных задач:
\begin{enumerate}
    \item выполнен анализ предметной области. Сформированы строгие критерии
          к разрабатываемой системе, включающие прозрачность вычислений,
          наличие программного интерфейса, возможность локального хранения
          и безопасность. Выделены восемь ключевых сущностей и формализована
          математическая методология вычисления интегральных метрик, включая
          эффективный процент с игры, долю владений в атаке и рейтинги
          эффективности;
    \item спроектирована реляционная схема данных, приведённая к третьей
          нормальной форме и состоящая из восьми таблиц. Целостность фактов
          обеспечена на уровне декларативных ограничений и процедурных
          триггеров. Реализована ролевая модель на уровне системы управления
          базами данных, гарантирующая изоляцию аналитических запросов
          от административных операций;
    \item разработан серверный слой приложения, предоставляющий сетевой
          интерфейс из двадцати аналитических маршрутов. Для минимизации
          задержек спроектирована подсистема кэширования с алгоритмом
          префиксной инвалидации ключей, предотвращающим лавинообразные
          нагрузки на базу данных при массовом устаревании кэша;
    \item выполнена физическая реализация системы. Хранилище наполнено
          репрезентативной выборкой исторических данных объёмом более
          ста пятидесяти тысяч фактов. Регрессионное тестирование доказало
          математическую точность внутрибазовых вычислений: результаты работы
          хранимых процедур полностью совпали с эталонным независимым
          расчётом;
    \item проведено исследование производительности системы при различных
          сценариях нагрузки. Доказано, что применение денормализованной
          витрины данных с составным древовидным индексом ускоряет
          аналитические агрегации в~61~раз. Интеграция подсистемы
          кэширования снижает медианное время ответа сервера
          в~3{,}8~раза. Нагрузочное тестирование показало, что при нагрузке
          до 50~одновременных подключений система обеспечивает P95 менее
          100~мс; при 100~подключениях рекомендуется горизонтальное
          масштабирование API-уровня.
\end{enumerate}

Разработанный программный комплекс представляет собой готовое
масштабируемое решение, применимое для обеспечения деятельности
аналитических отделов спортивных организаций.
